What Is Pardot?

Pardot — officially rebranded as Salesforce Marketing Cloud Account Engagement — is MMA's primary email marketing platform. Every mass email sent to members (event invitations, newsletters, webinar promotions, research alerts) goes through Pardot.

Pardot works with Salesforce contacts through a concept called Prospects. Every mailable contact in Salesforce has a corresponding Prospect record in Pardot. The two systems sync automatically and continuously.

📋
Lists
Groups of prospects you want to email. Can be static (manual) or dynamic (rule-based). The backbone of every send.
✉️
List Emails
The actual emails you send to a list. Built from a template, sent to one or more lists at a scheduled time.
🖼️
Templates
Reusable email designs. Build once, use many times. Contain the layout, branding, and variable merge fields.
📝
Form Handlers
Connect external web forms to Pardot. When someone submits a form on MMAGlobal.com, Pardot captures it and can trigger actions.
⚙️
Automation Rules
Triggered actions based on prospect criteria. "If prospect is in list X and score > 50, add to list Y." Run once or repeatedly.
🔄
Engagement Studio
Visual drip-sequence builder. Send a series of emails over time with branching logic based on how prospects respond.

Salesforce ↔ Pardot Sync

Understanding the sync is critical. Changes in one system propagate to the other — but there are rules about which system "wins" when there's a conflict.

How the Sync Works

  • Every Salesforce Contact with an email address can sync to a Pardot Prospect. Contacts missing an email address are not synced.
  • The sync runs continuously (near real-time). New Contacts created in Salesforce appear in Pardot within minutes.
  • Salesforce is the system of record for identity data (name, email, company, title). Pardot wins on engagement data (score, grade, opens, clicks).
  • If HasOptedOutOfEmail = true in Salesforce, Pardot will also suppress that prospect. The opt-out syncs both directions.
  • If a Pardot email hard bounces, Pardot sets IsEmailBounced = true in Salesforce automatically.
DirectionWhat Syncs
SF → PardotContact name, email, title, company, opt-out status, account membership status
Pardot → SFEmail bounce status, engagement score (pi__score__c), grade (pi__grade__c), last activity (pi__last_activity__c), hard bounce flag (pi__pardot_hard_bounced__c)
Both directionsOpt-out status (HasOptedOutOfEmail) — change in either system propagates to the other

Lists

Lists are the foundation of Pardot sending. Every email goes to one or more lists. Understanding the different list types prevents you from accidentally emailing the wrong people.

List Types

TypeHow It WorksWhen to Use
Static List A fixed set of prospects you manually add or import. Doesn't change automatically. One-off sends, event invitation lists, imports from CSV or Salesforce reports
Dynamic List Automatically updates based on criteria you define (e.g., "all prospects where Account Status = Active-Parent and Region = NA"). Membership changes as records change. Ongoing sends to rolling audiences (e.g., all mailable Marketer members)
Suppression List A list of prospects to exclude from a send. Checked against the mailing list before delivery. Excluding already-registered contacts, opt-outs, specific companies, or Outreach sequence prospects
Public List Visible in the Email Preference Center — prospects can self-subscribe or unsubscribe from these. Newsletter subscriptions, program-specific opt-ins

Creating a Static List

  • 1
    Navigate to Prospects > ListsIn Pardot, go to the Lists menu under the Prospects tab.
  • 2
    Click "+ Add List"Give the list a clear, descriptive name. Follow the MMA naming convention: include the event/program name and date (e.g., CCS 2026 — Round 1 Invites).
  • 3
    Set the list typeLeave "Is Public" unchecked unless this list should appear in the Email Preference Center. Leave "Is CRM Visible" checked so the list appears on Prospect records in Salesforce.
  • 4
    Add prospectsUse the "Add Prospects" button to search and add by email, or import a CSV. Alternatively, run a Pardot Automation Rule or use the Python script to batch-add by Salesforce Contact ID.

Creating a Dynamic List

  • 1
    Go to Prospects > Lists > "+ Add List"Check the "Dynamic List" checkbox. This reveals the criteria builder.
  • 2
    Set criteriaAdd rules based on Prospect fields, CRM fields (Salesforce fields), or activity. Example: Prospect CRM field "Account Status" contains "Active" AND Prospect CRM field "Region__c" equals "NA".
  • 3
    Save and verify countPardot will calculate the current list size. Verify the count looks right before using the list in a send. Dynamic lists update automatically as records change.
⚠️
Dynamic lists don't apply MMA's full mailable definition automatically. Pardot can filter on CRM fields, but it can't replicate the full SOQL logic (active account, region, board exclusions, email bounced, opted out) in one dynamic list. For complex audience pulls, it's more reliable to build the list in Salesforce/Python, export the emails, and import to a static Pardot list. Talk to Aaron Sultan if you need a segmented pull.

Email Templates

Templates are reusable email layouts. Jason and Dan maintain MMA's template library. A template defines the visual structure — header, body regions, footer, unsubscribe link — and uses merge fields to personalize content at send time.

Creating or Editing a Template

  • 1
    Go to Content > Email TemplatesBrowse existing templates. MMA has templates for event invites, newsletters, webinar promos, and member communications.
  • 2
    Clone before editingNever edit an active template directly. Clone it first (Actions > Copy), then edit the copy. This preserves the original for historical sends.
  • 3
    Use the drag-and-drop editor or HTMLPardot's builder lets you drag content blocks. For more control, switch to the HTML editor. MMA templates use inline CSS for email client compatibility.
  • 4
    Add merge fields for personalizationUse Pardot merge syntax: {{Recipient.FirstName}} for first name, {{Recipient.Email}} for email. Variable tags are available for any Prospect field synced from Salesforce.
  • 5
    Send a testAlways send a test to yourself and at least one other person before saving as final. Check rendering in Gmail, Outlook, and on mobile.

Every email must include an unsubscribe link. Use %%unsubscribe%% in your template footer. Pardot will block the send if this is missing — and omitting it also violates CAN-SPAM law.

Sending a List Email

A List Email is a one-time send to one or more lists. This is the most common send type at MMA — event invitations, webinar promotions, member announcements, etc.

Step-by-Step: Send a List Email

  • 1
    Go to Emails > List Emails > "+ Add List Email"This opens the email creation wizard.
  • 2
    Name the emailUse a clear internal name that includes the event/program and send date. Example: CCS 2026 — Round 1 Invite — 2026-04-15. This name is internal only and never seen by recipients.
  • 3
    Select your templateChoose the appropriate pre-built template. If you need a new design, build the template first, then come back here.
  • 4
    Fill in email detailsSet the Subject line, From name (e.g., "MMA Global"), and From email address (must use an MMA domain). Add a preheader text — this appears in the inbox preview after the subject line.
  • 5
    Add sending list(s)In the Recipients section, add one or more mailing lists. If sending to multiple lists, Pardot deduplicates automatically — a prospect on two lists only gets the email once.
  • 6
    Add suppression list(s)This is critical. Add any suppression lists needed — prospects already registered, contacts who should be excluded, or Outreach sequences in progress. Suppression happens before send.
  • 7
    Send a testUse the "Send Test" button. Send to yourself and ideally one other person. Review the final look in your actual email client before proceeding.
  • 8
    Schedule or send immediatelyChoose a send time (usually business hours) or send immediately. Once sent, the email cannot be recalled — double-check everything before hitting confirm.
⚠️
Always check your suppression lists before sending. If you're sending a CCS invite and some contacts are already Registered in the Salesforce campaign, you want to exclude them. The best practice is to pull the already-registered list from Salesforce and upload it as a Pardot suppression list before the send. Aaron Sultan can provide this list.

Connecting Pardot Sends to Salesforce Campaigns

Every Pardot email send can — and should — be connected to a Salesforce Campaign. This is how email engagement (opens, clicks) flows into Salesforce as CampaignMember records, enabling reporting and engagement tracking.

How to Link a List Email to a Salesforce Campaign

  1. When creating the List Email, scroll to the Salesforce Campaign field in the email setup.
  2. Search for and select the corresponding Salesforce Campaign (e.g., CMO CEO Summit 2026_CCS 2026).
  3. Set the Member Status for each interaction type:
    • Email opened → typically "Sent" or leave as default
    • Email clicked → typically "Responded" or "Invited"
  4. Save. When the email sends, Pardot will create or update CampaignMember records in Salesforce based on engagement.
💡
Campaign connection = engagement tracking. Without linking to a Salesforce Campaign, email opens and clicks stay in Pardot only and don't appear on the Contact record or in Salesforce reports. Always link your sends to the corresponding campaign.

Form Handlers

A Form Handler connects an external web form (on MMAGlobal.com or any landing page) to Pardot. When someone submits the form, Pardot captures the submission, creates or updates the Prospect record, and can trigger automated actions.

Form Handlers are different from Pardot Forms — a Form is hosted and served by Pardot, while a Form Handler captures submissions from a form you already have on your own website.

Creating a Form Handler

  • 1
    Go to Content > Form Handlers > "+ Add Form Handler"Give it a descriptive name that matches the form it's connected to.
  • 2
    Configure field mappingsMap the form fields to Pardot Prospect fields. For example: form field "email" → Prospect field "Email", "first_name" → "First Name", "company" → "Company". The form field names must match exactly what the web form submits.
  • 3
    Set success/error locationsThe URL to redirect to after a successful submission (e.g., a "Thank You" page) and after an error.
  • 4
    Add Completion ActionsDefine what happens when someone submits. Common actions: add to a list, notify a user by email, add to a Salesforce Campaign, set a field value, assign to a user.
  • 5
    Copy the endpoint URLPardot gives you a unique endpoint URL. The web developer needs to point the form's action to this URL. This is what makes submissions flow into Pardot.

Completion Actions

Completion Actions fire immediately when a form is submitted. They are the most direct way to trigger Pardot and Salesforce updates from a form fill.

Completion ActionWhat It Does
Add to ListAdds the submitting prospect to a specified Pardot list
Remove from ListRemoves from a list (useful for unsubscribe forms)
Add to Salesforce CampaignCreates a CampaignMember record in Salesforce with a specified status (e.g., "Registered")
Notify UserSends an email alert to an MMA team member (e.g., notify Jason when someone registers)
Set Prospect Field ValueUpdates a field on the Prospect/Contact (e.g., set Source__c = 'Web Form')
Assign Prospect to UserAssigns the prospect to a specific Salesforce user for follow-up
Send Autoresponder EmailImmediately sends a confirmation or welcome email to the submitter
Increase ScoreBoosts the prospect's Pardot engagement score
Add TagTags the prospect for easy filtering later

Automation Rules

Automation Rules run on a schedule and apply bulk actions to prospects that match defined criteria. Unlike Completion Actions (which fire once on a specific event), Automation Rules continuously check your prospect database and apply actions to any matching records.

SettingOptions & Notes
Run typeRepeat — runs continuously, re-evaluating prospects on an ongoing basis. Once — applies the rule to each prospect only one time.
CriteriaAny combination of Prospect fields, CRM fields, activity, list membership, score, grade, etc.
ActionsSame set as Completion Actions: add/remove from lists, add to SF Campaign, notify users, set fields, etc.

Example: Auto-Add Registered Contacts to a Suppression List

Criteria: Prospect CRM field "CampaignMember.Status" = "Registered" AND CampaignMember.CampaignId = [CCS 2026 Campaign ID]
Action: Add to list "CCS 2026 — Already Registered (Suppress)"
Run type: Repeat
Effect: As people register through the event website, they're automatically added to the suppression list and won't receive future invite sends.

⚠️
Automation Rules can run on your entire prospect database. Before enabling a rule, always preview how many prospects match the criteria. Accidentally adding 50,000 prospects to the wrong list is hard to undo.

Engagement Studio

Engagement Studio is Pardot's visual drip-sequence builder. Use it when you want to send a series of emails over time with branching logic — for example, send a webinar reminder, wait 3 days, check if they opened it, then send a different follow-up based on whether they did or didn't.

When to Use Engagement Studio vs. List Emails

Use List Email when...Use Engagement Studio when...
Sending a single one-time email to a listSending a sequence of 2+ emails over days or weeks
A simple blast with no follow-up logicYou need "if they opened → do X, if they didn't → do Y" branching
Speed matters — setting up is fasterNurture sequences, onboarding flows, multi-touch event promotion

Building an Engagement Studio Program

  • 1
    Go to Automation > Engagement Studio > "+ Add Program"Name the program clearly. Set the recipient list(s) and any suppression lists upfront.
  • 2
    Build the flowDrag-and-drop elements: Send Email (pick a template), Wait (pause X days), Trigger (did they open? click? visit a page?), Action (add to list, update a field, notify a user).
  • 3
    Test the programUse the "Test" tab to walk through the flow with a test prospect and verify each branch fires correctly.
  • 4
    Start the programOnce live, prospects on the recipient list begin moving through the sequence. New prospects added to the list enter the sequence at the beginning.

List Scoring, Rule Groups & Segmentation Rules

Beyond basic list types, Pardot has a built-in scoring system and advanced criteria-building tools. Understanding these makes you a much more precise sender.

Prospect Scoring

Every prospect in Pardot has a score that increases as they engage with MMA's content. Higher score = more engaged contact.

ActionPointsNotes
Email click+3 ptsAny link click in a Pardot-sent email
File / content access+3 ptsAccessing a Pardot-tracked file download or landing page
Form submission+10 ptsSubmitting a Pardot form or form handler
Email open+1 ptLess reliable due to Apple/Gmail image blocking — clicks are a stronger signal
Page visit+1 ptVisiting a Pardot-tracked landing page
💡
Scoring categories are scoped to folders. Rather than relying on the global score alone, Pardot's scoring categories let you track engagement per initiative — e.g., a "CCS 2026" scoring category that only tallies activity from CCS-related email and form assets. This is more useful than the global score when evaluating event-specific engagement or comparing across programs.

Rule Groups in Dynamic Lists

When building dynamic list criteria, Pardot uses AND logic within a rule group and OR logic between groups. This lets you build complex "this AND that, OR this other combination" audiences without splitting into multiple lists.

Example: Multi-Condition Dynamic List

Goal: All mailable NA Marketer members, OR any board member who is also a Marketer

Rule Group 1 (AND logic inside the group):

  • Account Status = Active-Parent
  • Primary Ecosystem = Marketer
  • Region = NA

Rule Group 2 (OR from Group 1 — AND logic inside):

  • Is Board Member = true
  • Primary Ecosystem = Marketer

Result: Any prospect who satisfies Group 1 OR Group 2 is included.

Segmentation Rules

Segmentation Rules are similar to Automation Rules, but they run only once. Use them when you want to perform a one-time bulk action on a defined segment — for example, adding all contacts who clicked a specific link in a past email to a new list. Once run, they don't re-evaluate new prospects. Segmentation Rules are useful for historical cleanup and one-time list population.

Disengaged Contacts & Deliverability

One of the most important operational concepts in Pardot: disengaged contacts — prospects who haven't interacted with an email in 6 or more months — require careful handling to protect MMA's sender reputation and overall email deliverability.

Prospect StatusDefinitionRisk if Ignored
EngagedHas opened, clicked, or otherwise interacted with a Pardot email within the last 6 monthsLow — safe to email through the normal Pardot environment
UnengagedNo email engagement activity for 6+ monthsHigh — mass-emailing unengaged contacts can tank MMA's sender reputation
🚫
Do not mass-email large lists of unengaged contacts from MMA's main Pardot. ISPs interpret sending to many disengaged contacts as a signal that you're emailing people who don't want your mail. This drives up spam complaints, damages MMA's sender reputation score, and can cause all future emails — even to engaged contacts — to land in spam or get blocked. Screen your lists for engagement before large sends.
💡
Audit list engagement before every large send. Before sending to a list of 5,000+ contacts, check when those prospects last engaged. Pardot's prospect list view can be filtered by last activity date. If a significant portion of your list hasn't engaged in 6+ months, consider a smaller re-engagement send first rather than blasting the full list.

Email Best Practices

Apply these best practices to every MMA email campaign, regardless of audience size.

Subject Lines

  • 50–65 characters is the sweet spot — displays fully in most desktop and mobile clients without truncation
  • Front-load the key message — the most important word should come first, not last, since mobile clients truncate early
  • Personalize when possible — adding the recipient's first name or company can lift open rates
  • Avoid spam-trigger words — FREE, URGENT, CLICK NOW, excessive ALL CAPS, multiple exclamation points (!!!)
  • Test before committing — when Pardot allows A/B testing, test one variable at a time (emoji vs. no emoji, question vs. statement)

Pre-Header Text

The pre-header is the preview snippet that appears after the subject line in most inboxes. It should complement the subject — not repeat it. Keep it under 100 characters. If you leave it blank, email clients pull the first visible line of your email body (which is often the unsubscribe link or an image alt tag).

Content Length

Email TypeTarget Word CountWhy
Event invite / cold outreach100–150 wordsGet to the point. Busy executives won't scroll through a wall of text.
Newsletter / member update250–300 wordsSubscribers expect more content — still keep it scannable with headers and bullets.
Reminder / follow-up50–100 wordsShorter is better for reminders — just the CTA and key details.

Responsive Design (Mobile-First)

  • Single-column layout — multi-column side-by-side designs break on mobile and should be avoided
  • Minimum 14px font size for body text — anything smaller is hard to read without zooming on a phone
  • CTA buttons at least 44px tall — needs to be thumb-friendly on touchscreens
  • Images under 200kb — large images load slowly on mobile data; always include descriptive alt text in case images are blocked
  • Never send an email that's only images — spam filters will block it, and contacts with image loading off will see nothing

Calls to Action (CTAs)

  • One primary CTA above the fold — visible without scrolling; this is the action you most want them to take
  • One secondary CTA in the body — reinforces the action for readers who scroll down
  • Use styled button elements, not plain text links, for the primary CTA — they're more visible, more clickable, and easier to track
  • Action-oriented CTA text — "Register Now", "View the Report", "Save Your Seat" (not just "Click Here" or "Learn More")

Segmentation & Personalization

  • Segment before you send — a Marketer member and a Solution Provider should receive tailored messaging, not the same generic blast; the same event appeals differently to each ecosystem
  • Use variable tags for personalization: {{Recipient.FirstName}}, {{Recipient.Company}}, etc.
  • Always test merge fields — send a test to an account with known field values to confirm tags populate correctly before the live send
  • Set fallback values — if a contact has no first name on file, the fallback should be something graceful like "Hi there" rather than a blank space

Completion Actions

Completion Actions fire automatically when a prospect takes a specific action (clicks a link, opens an email, submits a form). They're one of the most useful Pardot tools for post-send automation.

TriggerCommon Completion Actions
Email clickAdd to list, add to Salesforce Campaign (with status "Responded"), change a field value, notify a user
Email openAdd to list, increase score — less reliable as a trigger since opens are often miscounted
Form submissionAdd to list, add to SF Campaign (with status "Registered"), send autoresponder, set field values
💡
Campaigns are managed in Salesforce, not Pardot Classic. When using Completion Actions to add a prospect to a Salesforce Campaign, always make sure the Campaign already exists in Salesforce (create it there first). Pardot completion actions reference Salesforce Campaign IDs — they don't create campaigns on their own.

Send Timing

  • Best days: Tuesday, Wednesday, Thursday
  • Best time: 9–11am Eastern — catches East Coast at the start of the workday while still being early enough on the West Coast
  • Avoid: Mondays (post-weekend inbox overload), Fridays (people checking out early), major US holidays and holiday windows
  • Global sends: NA BU uses Eastern as the baseline; EMEA and APAC sends go through their respective BUs with locally appropriate timing managed by those teams

Compliance

Every email MMA sends must comply with applicable email marketing laws. Three regulations govern MMA's audience: CAN-SPAM (US), GDPR (EU/UK), and CASL (Canada). All three require the same core behaviors — an easy opt-out, accurate sender identity, physical address, and prompt honoring of unsubscribes.

🇺🇸
CAN-SPAM (US)
Requires a clear unsubscribe mechanism, physical mailing address, non-deceptive subject lines, and accurate From/Reply-To addresses. Honor opt-out requests within 10 business days.
🇪🇺
GDPR (EU/UK)
Requires legitimate interest or explicit opt-in consent to email EU/UK contacts. Must honor right-to-erasure requests. MMA's EMEA Pardot BU is separate partly for GDPR compliance — EU contacts should route through EMEA.
🇨🇦
CASL (Canada)
Stricter than CAN-SPAM. Requires express or implied consent. Implied consent expires (typically 2 years from last transaction). Canadian contacts require extra care — when in doubt, don't send through the NA BU.
RequirementCAN-SPAMGDPRCASL
Unsubscribe / opt-outRequiredRequiredRequired
Honor opt-out within10 business daysAs soon as possible10 business days
Physical address requiredYesYesYes
Accurate From / Reply-ToYesYesYes
Consent required before sendingNo (opt-out model)Yes (opt-in)Yes (express or implied)
🚫
The %%unsubscribe%% tag is non-negotiable. Every email template must include it in the footer. Pardot blocks sends that don't have an unsubscribe link — and omitting it violates all three laws above. If you're building a new template, add the unsubscribe link from the very start.
⚠️
EMEA and APAC sends go through separate Pardot Business Units. GDPR compliance for EU contacts and CASL compliance for Canadian contacts is managed through separate BUs with their own consent frameworks. Never add EU or Canadian contacts to the NA BU for a send without checking with Aaron Sultan first.
